# Contributing to Wasfatna

Thank you for your interest in contributing to Wasfatna! We welcome contributions from everyone to help make this recipe platform better for the community.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub with:
- A clear description of the problem.
- Steps to reproduce the issue.
- Expected vs. actual behavior.

### Suggesting Features
We love new ideas! If you have a feature request, please open an issue to discuss it first.

### Pull Requests
1. **Fork the Repository**: Create your own fork of the project.
2. **Create a Branch**: Create a feature branch (`git checkout -b feature/amazing-feature`).
3. **Commit Your Changes**: Make small, frequent commits with clear messages.
4. **Push to GitHub**: Push your branch to your fork (`git push origin feature/amazing-feature`).
5. **Open a Pull Request**: Submit a PR to the `main` branch of the original repository.

## Coding Standards

### Backend (Laravel)
- Follow [PSR-12](https://www.php-fig.org/psr/psr-12/) coding standards.
- Write meaningful variable and method names.
- Ensure all new features have corresponding tests if possible.
- Use Laravel's built-in features (Eloquent, Collections, etc.) where appropriate.

### Frontend (Vue 3 / VueUse / Tailwind)
- Use Vue 3 Script Setup syntax.
- Use Tailwind CSS for styling.
- Ensure components are reusable and well-organized in `resources/js/components`.
- Use Typescript for better type safety.

## Local Development Setup
Please refer to the [README.md](README.md) for detailed instructions on setting up your local environment.

## Code of Conduct
Please be respectful and helpful to others in the community. We aim to maintain a welcoming environment for everyone.
